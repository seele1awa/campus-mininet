# 基于 Mininet 的校园网构建课程项目报告

## 1. 项目背景

校园网需要同时服务学生宿舍、教学楼、图书馆、办公楼以及行政部门。不同区域的业务需求不同：普通用户需要稳定访问校内 Web/FTP 资源，行政部门需要更高的访问控制要求，网络还需要具备一定扩展能力，便于后续增加实验室、无线接入或访客网络。

本项目使用 Mininet 搭建校园网仿真环境，通过 Linux 路由节点、Open vSwitch 接入交换机和 `iptables` 安全策略实现校园网的通信、资源共享和访问控制。

## 2. 项目目标

1. 实现校园网各节点之间按需求通信，保证部门内部和部门之间的数据传输能力。
2. 在核心路由节点部署 ACL 安全策略，保护人事处、财务处等敏感区域。
3. 采用模块化地址规划和分层拓扑，为后续新增区域和设备预留空间。

## 3. 需求分析

课程题目提出三项功能要求：

1. 每个部门内部实现二层互通，部门间实现三层互通。
2. 所有用户均可在内网实现资源共享，访问 Web/FTP 服务器。
3. 特定区域，如人事处、财务处，需要设置访问控制，限制其他区域访问。

根据这些要求，网络被划分为多个独立网段，每个区域使用一台接入交换机实现二层转发，不同区域通过核心路由器进行三层转发。安全策略集中放置在核心路由器上，便于统一管理和扩展。

## 4. 总体设计

项目采用“核心层 + 接入层 + 服务器区”的结构：

```text
                  attacker1
                      |
                    s_ext
                      |
                   r_core
   -------------------------------------------------
   |       |        |        |       |       |      |
 s_stu  s_teach   s_lib  s_office  s_hr   s_fin  s_srv
   |       |        |        |       |       |      |
 宿舍区   教学楼    图书馆    办公楼   人事处  财务处  Web/FTP
```

设计特点如下：

- 每个区域独立子网，广播域隔离清晰。
- 每个区域通过一台接入交换机连接终端。
- 核心路由器 `r_core` 负责跨区域三层转发。
- 服务器区独立成网段，部署 Web 和 FTP 服务。
- 外部模拟区用于验证外部访问阻断策略。

|---|---|---|---|

| 教学楼 | `10.10.20.0/24` | `10.10.20.1` | `teach1`, `teach2` |
| 图书馆 | `10.10.30.0/24` | `10.10.30.1` | `lib1`, `lib2` |
| 办公楼 | `10.10.40.0/24` | `10.10.40.1` | `office1`, `office2` |

| 财务处 | `10.10.60.0/24` | `10.10.60.1` | `fin1` |

地址规划使用 `10.10.X.0/24` 的形式，后续可继续扩展 `10.10.70.0/24`、`10.10.80.0/24` 等区域。

## 6. Mininet 实现

项目核心脚本为 `src/campus_net.py`。脚本主要包含以下模块：

1. `CampusTopo`：定义校园网拓扑结构，为每个区域创建交换机和主机。
2. `LinuxRouter`：定义核心 Linux 路由节点，开启 IPv4 转发。
3. `configure_security`：在核心路由器上配置 `iptables` ACL。
4. `start_services`：在服务器区启动 Web 和 FTP 服务。
5. `run_tests`：执行自动化连通性和安全策略测试。

交换机使用 Open vSwitch 的 standalone 模式，保证在没有额外控制器进程时也可以完成二层交换。跨网段通信由 Linux 路由节点完成。

运行方式：

```bash
sudo ./run_campus_net.sh
```

自动测试方式：

```bash
sudo ./run_campus_net.sh --test
```

## 7. Web/FTP 服务设计

服务器区包含两个服务节点：

- `web`：地址 `10.10.100.10`，使用 Python HTTP Server 提供 Web 页面。
- `ftp`：地址 `10.10.100.20`，使用项目内置的标准库 FTP 服务提供文件下载。

FTP 服务没有依赖第三方 Python 包，方便在普通 WSL Ubuntu 环境中运行。

测试命令：

```bash
stu1 curl -s http://10.10.100.10
stu1 curl -s ftp://10.10.100.20/README.txt
```

## 8. 安全策略设计

安全策略部署在 `r_core` 上，使用 `iptables` 的 `FORWARD` 链实现。

主要规则如下：

1. 外部模拟区 `203.0.113.0/24` 禁止访问校园内网 `10.10.0.0/16`。
2. 学生宿舍、教学楼、图书馆禁止访问人事处和财务处。
3. 办公楼允许访问人事处和财务处。
4. 内网用户允许访问服务器区的 Web/FTP 服务。
5. 普通非敏感区域之间保持三层互通。

对应代码逻辑：

```python
normal_user_subnets = ["10.10.10.0/24", "10.10.20.0/24", "10.10.30.0/24"]
sensitive_subnets = ["10.10.50.0/24", "10.10.60.0/24"]
for source in normal_user_subnets:
    for dest in sensitive_subnets:
        router.cmd(f"iptables -A FORWARD -s {source} -d {dest} -j REJECT")
```

该策略符合最小权限原则：普通区域不能访问敏感部门，但办公业务需要的访问路径被保留。

## 9. 测试方案

| 测试项 | 命令 | 预期结果 |
|---|---|---|
| 宿舍区内部二层互通 | `stu1 ping -c 2 stu2` | 成功 |
| 宿舍区到教学楼三层互通 | `stu1 ping -c 2 teach1` | 成功 |
| 学生访问 Web 服务 | `stu1 curl -s http://10.10.100.10` | 成功 |
| 学生访问 FTP 服务 | `stu1 curl -s ftp://10.10.100.20/README.txt` | 成功 |
| 学生访问人事处 | `stu1 ping -c 2 hr1` | 失败 |
| 教学楼访问财务处 | `teach1 ping -c 2 fin1` | 失败 |
| 办公楼访问人事处 | `office1 ping -c 2 hr1` | 成功 |
| 办公楼访问财务处 | `office1 ping -c 2 fin1` | 成功 |
| 外部主机访问服务器区 | `attacker1 ping -c 2 web` | 失败 |

脚本提供 `--test` 参数自动执行上述测试，并输出通过情况。

## 10. 实时联动可视化展示

项目提供 `src/campus_live_server.py` 和 `visualization/index.html` 作为答辩展示系统。实时控制台运行后，浏览器页面通过 HTTP API 与 WSL 中的 Mininet 后端通信，不再只是前端静态模拟。

启动方式：

```bash
sudo ./run_live_server.sh 8088
```

浏览器访问：

```text
http://127.0.0.1:8088
```

界面包含：

- 校园网拓扑图。
- 节点 IP、区域、网关、服务信息。
- 真实拓扑启动和停止按钮。
- 源节点、目标节点和操作类型选择。
- 真实 `ping`、Web 访问、FTP 下载结果。
- 可选择消息模板的主机间 TCP 消息发送。
- 目标主机实际收到的消息 payload。
- ACL 判定结果、命令输出和路径高亮。
- 典型测试场景快速切换。

消息发送过程为真实网络通信：后端先在目标 Mininet 主机上启动一次性 TCP 接收器，再由源 Mininet 主机连接目标 IP 和端口发送 JSON 消息。若路径被 ACL 阻断，消息无法到达目标主机，页面会显示发送失败。

实时控制台还提供一键自动测试，除原有连通性和服务访问测试外，额外验证：

1. 办公楼 `office1` 向财务处 `fin1` 发送业务消息成功。
2. 学生宿舍 `stu1` 向人事处 `hr1` 发送消息被 ACL 拦截。

## 11. 可扩展性分析

本设计具备较好的扩展能力：

1. 新增区域时，只需在 `AREAS` 配置中增加网段、交换机和主机。
2. 新增服务器时，可直接加入服务器区或新增独立服务区。
3. 安全策略集中在核心路由器，便于统一调整。
4. 地址规划保留了多个未使用网段，便于后续接入无线网、实验室或访客网络。
5. 可进一步接入 Ryu、POX 等 SDN 控制器，实现更细粒度的流表控制。

## 12. 总结

本项目基于 Mininet 完成了校园网仿真系统，实现了部门内部二层互通、部门间三层互通、Web/FTP 资源共享和敏感区域访问控制。项目同时提供自动化测试脚本和实时联动可视化控制台，能够较完整地展示校园网的拓扑结构、地址规划、路由通信、安全策略和主机间业务消息传输过程。
